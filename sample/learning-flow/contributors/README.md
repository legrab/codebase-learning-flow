# Contributors

Each contributor receives a compact personal workspace:

```text
contributors/<slug>/
├── README.md
├── TRACKING.md
├── themes/
└── ticket-paths/
```

## Identity

Before the first personal write:

1. inspect Git name and email;
2. infer a filesystem-safe stable slug;
3. ask the contributor to confirm it;
4. create the directory only after confirmation.

Do not store unnecessary personal information.

## Personal files

Personal files should contain:

- current objective;
- compact prior takeaways;
- contributor questions and answers;
- unresolved uncertainty;
- tailored follow-up;
- progress;
- demonstrated applications.

They should not duplicate the complete shared material.

After evaluation, normalize raw notes and compact resolved interaction history.

## Shared improvement

When contributor confusion reveals a reusable weakness, improve shared research or material and keep the personal trace minimal.
