# Optional contributor traces

Shared repository learning does not require identity.

Create `contributors/<slug>/` only when the user explicitly requests personal tracking, a durable personal theme, or a ticket-specific learning trace. Ask once for a filesystem-safe slug at that point.

Personal files are working surfaces, not diaries. Keep evidence, durable takeaways, useful uncertainty, and next challenge. Do not maintain scores or transcripts. Promote reusable verified findings to the shared canonical surface and link back instead of copying them.
